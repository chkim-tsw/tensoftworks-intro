1 사업 이해도

프로젝트 본질

겉으로는 하나의 쇼핑몰이지만, 안에는 세 가지 완전히 다른 비즈니스 모델이 돌아갑니다.

유통 (원자재)

본사가 직접 팔고, 직접 배송하고, 직접 CS. 정산도 없는 자사몰 구조. 다만 향후 입점 기능을 열 수 있게 설계만 열어두라는 조건 부여.

판촉 (판촉물)

입점사가 상품 올리고, 주문 받고, 배송하고, CS 1차 대응. 본사는 수수료 떼고 정산해주는 마켓플레이스 구조.

실사출력

판촉과 동일한 입점 구조인데, 주문제작 방식이라 사이즈 직접 입력, 디자인 파일 업로드, 실시간 견적 계산 추가.

이 프로젝트의 난이도는 결국 "하나의 결제에서 시작된 주문이 판매자별로 갈라져서 각각의 배송 → CS → 구매확정 → 정산 → 세금계산서 → ERP까지 흘러가는 전체 파이프라인"을 얼마나 깔끔하게 설계하느냐에 달려 있습니다.

2 기능별 설계 방향

2.1 상품등록 체계

판촉물 특성상 옵션 조합이 곱하기로 늘어나는 구조입니다. 인쇄 방식, 소재, 수량 구간별 단가, 사이즈, 색상 — 이게 다 조합이니까 가격 테이블이 매트릭스가 됩니다.

KT alpha 판촉몰에서 이미 구현한 방식을 기반으로 설계합니다.
셀러센터(SCM)에서 상품 등록 시 기본정보 탭과 공급정보 탭을 분리합니다. 카테고리는 다중 추가가 가능하고, 썸네일 이미지(500×500 권장, 최대 3개), 법적 허가사항, 재질, 규격을 입력하는 영역을 둡니다. 공급가액은 VAT 별도 기준으로 입력하고, 제작 기간과 배송비 부담 주체(셀러/본사)를 선택합니다.

옵션 설정

인쇄/박스/포장 각각에 대해 사용/미사용을 선택하고, 사용 시 가격 방식을 무료 / 통합(총액) / 개당 중에서 선택하는 구조입니다. 이 방식이 판촉물 옵션 매트릭스를 가장 현실적으로 처리하는 UI이고, KT alpha에서 셀러들이 실제로 사용하면서 안정화된 패턴입니다.

칼선(디자인) 등록 — 실사출력 전용

25MB 이하 jpg/png 칼선 이미지를 첨부하고, 팝업 에디터에서 마우스로 인쇄 가능 영역을 직접 설정합니다. 실제 제작용 AI/PDF 파일은 별도 업로드 버튼으로 분리합니다. RFP의 자체 에디터 API 연동(FR-SHP-007)과 정확히 맞는 구조입니다.

상품 상태 관리

임시저장→ 승인대기→ 1차승인→ 최종승인

승인대기 상태에서는 셀러가 상품 정보를 수정할 수 없고, 거절/보류 시 사유를 확인한 후 보류 건에 한해 재심사 요청이 가능합니다. 대용량 엑셀 등록도 지원합니다.

관리자(BO) 측 핵심: 수량별 마진율 설정

셀러가 등록한 공급가 위에 관리자가 최소/최대 수량 기준으로 마진율을 책정하면 소비자가격이 자동 계산됩니다. 최종 승인 후에는 마진율 수정이 불가능하게 잠급니다. 이 구조가 입점몰에서 본사 수익을 관리하는 가장 깔끔한 방식이고, RFP의 카테고리별/입점사별 수수료율(FR-STL-002)과 함께 이중으로 수익 구조를 설계할 수 있습니다.

2.2 장바구니 → 결제 → 주문 분리

장바구니에 담기는 시점부터 판매자 구분값(본사/입점사ID)을 보유합니다. 결제는 PG 한 번에 처리하고, 내부에서 판매자별 주문을 자동 분리합니다. 배송비는 판매자별 개별 산정합니다. PG 수수료는 본사 부담으로 설계하고 수수료율에 반영합니다.

장바구니에서 결제로 넘어갈 때 판매자별 주문 분리 방식, 배송비 판매자별 부과, 합배송 가능 여부 등은 RFP에 명시되어 있지 않습니다. PG 한 번 결제 + 내부 분리 방식으로 설계하되, 발주사 확인 후 확정합니다.

2.3 주문 프로세스 및 상태 관리

셀러센터(SCM) 주문관리

화면 상단에 파이프라인 형태로 주문 단계를 배치합니다. 단계별 건수를 직관적으로 확인하고 필터링할 수 있는 구조입니다.

주문서 접수→ 주문확인→ 시안제작→ 시안확정→ 상품제작→ 배송중→ 배송완료

관리자(BO) 주문관리

'발주신청내역'부터 '1단계: 주문확인', '2단계: 상품제작', '3단계: 배송중', '4단계: 배송완료'까지 프로세스별로 목록을 조회합니다. 주문서 상세에서 공급가, 부가세, 결제 금액(PG/비즈머니), 배송 주의사항, 고객 시안 요청 내역(칼선 이미지, PDF/AI 파일)을 모두 확인할 수 있습니다.

핵심 비즈니스 룰: KT alpha 판촉몰 운영 경험에서 확인된 것으로, "상품제작중" 상태에서는 주문 취소가 불가능합니다. 주문제작 상품의 특성상 제작에 들어간 후에는 원상복구가 안 되기 때문입니다. 이 룰은 애드피아몰의 판촉/실사출력 카테고리에도 동일하게 적용되어야 합니다.

셀러-관리자 커뮤니케이션

주문 목록의 메모 기능으로 관리자↔셀러 간 실시간 소통을 지원합니다. 1:1 문의는 이벤트/정산/배송비 등 유형별 셀렉트 박스 + 텍스트 에디터 + 첨부파일 업로드로 구성하고, 답변 완료 시 '처리'로 변경, 추가 댓글 시 '추가처리'로 자동 변경되는 상태 관리를 적용합니다.

2.4 셀러 관리 및 권한 체계

셀러 가입 흐름

이메일 ID + 휴대전화 인증→ 사업자/회원 정보 입력→ 관리자 승인→ SCM 로그인 활성화

반려 시 사유가 셀러에게 그대로 노출되어 수정 후 재신청이 가능합니다. SRM 시스템 회원가입이 선행되어야 SCM 가입 심사가 가능한 구조는 RFP의 공급사 관리 시스템과 연결됩니다.

관리자 권한

BO 관리자별로 접근 가능한 메뉴(상품/주문/정산/전시 등)를 개별 부여/해제할 수 있게 합니다. 셀러 비밀번호 초기화, SCM 이용중지 처리가 가능하되, 이용중지 상태라도 FO 로그인 및 구매에는 영향을 주지 않습니다.

서브회원 관리

마스터 계정으로 서브회원의 ID/이름/전화번호를 등록하고, 토글 스위치로 이용중/이용정지를 제어합니다. B2B 입점사 특성상 담당자가 여러 명인 경우가 많으므로 필수 기능입니다.

2.5 견적서 → 주문 전환

견적서 발급→ 고객 승인→ 주문서 자동 생성→ 결제 (또는 후불 배정)→ 입점사/생산시스템 전달

견적 유효기간 설정이 가능하고, B2B 대량 주문은 엑셀 업로드 + 견적 일괄 생성으로 처리합니다.

후불결제 계약 관리: 후불결제 주문 건의 경우 법인 날인 기반의 계약서를 이메일로 발송하거나 PDF로 다운로드 및 인쇄하여 관리할 수 있는 기능을 포함합니다.

2.6 후불 / 여신한도

기업회원별 여신한도 설정(FR-B2B-004, FR-B2B-102), 주문 시 실시간 잔여한도 체크, 한도 초과 시 선결제 전환 또는 주문 차단. 월정산 시 미수금 자동 집계, 관리자 대시보드에서 기업별 채권 현황 조회.

기업고객 대상 여신 관리를 기본 범위로 설계하되, 입점사 간 채권 관리는 Phase 2로 분리하는 것을 권장합니다.

2.8 취소 / 반품 / 교환

입점사 1차 승인/거부→ 미응답 48시간→ 본사 자동 이관→ 분쟁 시 본사 중재

CS 메뉴에서 취소관리/반품관리/교환관리를 통합 관리합니다. KT alpha 경험에서 초기에는 "주문취소요청목록/반품완료목록/교환완료목록"으로 나눴다가, 실제 운영에서 "취소관리/반품관리/교환관리"로 메뉴명과 구조를 변경한 이력이 있습니다. 처음부터 상태 중심이 아니라 처리 유형 중심으로 메뉴를 잡겠습니다.

2.9 전시 / 기획전 / 이벤트 관리

카테고리 추가, 메인 배너 및 띠 배너 등록(PC/모바일 이미지, 시작/종료일 지정), 베스트100, MD PICK, 브랜드관 구성을 포함합니다. 브랜드 등록 시 백색 및 컬러 로고 이미지(최대 2MB 제한)를 업로드합니다.

이벤트/기획전은 할인 또는 증정 등의 이벤트를 기획하고 상세 배너 이미지를 등록하여 기간별로 노출합니다. KT alpha에서 초기 기획 대비 추가된 메뉴(브랜드관리/메인배너관리/MDPICK관리)들이 실제 운영에서 필수로 밝혀진 항목들이므로, 애드피아몰에도 초기부터 포함합니다.

2.10 무통장입금 처리

가상계좌 발급 후 입금 대기 상태, 재고 홀딩 72시간(설정 가능), 미입금 시 자동 취소. 입금 확인은 PG 자동 통보. 입점사 주문에 대한 무통장입금이면 입금 확인 주체와 주문 활성화 시점을 명확히 분리합니다.

3 의견이 필요한 영역

3.1 사방넷 연동 — 구조적 미스매치에 대한 의견

RFP의 FR-VND-003에 "재고 관리 — 재고 수량, 사방넷 연동"이 명시되어 있습니다. 그런데 여기에는 근본적인 구조 문제가 있습니다.

구분사방넷의 설계 철학애드피아몰이 필요한 구조
관계 구조 1 셀러 : N 쇼핑몰
하나의 판매자가 여러 채널 주문을 한 곳에서 관리		 N 셀러 : 1 쇼핑몰
여러 입점사가 하나의 몰에 상품을 올리는 마켓플레이스
재고 관리 SKU 기반 재고 추적이 핵심 판촉물/실사출력은 주문제작(make-to-order)이 대부분으로 SKU 재고 추적이 무의미

KT alpha 판촉몰에서도 재고 관리가 아니라 주문 상태 관리(주문확인→시안제작→시안확정→상품제작→배송)가 핵심이었습니다.

현실적 제안

  • 사방넷 연동이 실제로 필요한 입점사가 얼마나 되는지 발주사에 확인 필요
  • 유통(원자재) 카테고리의 재고 기반 상품을 취급하는 셀러에게는 의미가 있을 수 있으나, 판촉/실사출력 셀러에게는 사방넷보다 주문제작 프로세스 관리가 더 중요
  • 필수 요구사항이라면 사방넷 API는 부가서비스(유료) 별도 계약 필요, API 연동키도 셀러 각자가 발급
  • 연동 방향: 애드피아몰 → 사방넷으로 주문을 밀어넣는 구조가 되어야 함
사방넷 연동은 Phase 2로 분리하고, Phase 1에서는 자체 재고관리(유통 카테고리) + 주문제작 프로세스 관리(판촉/실사출력)로 충분히 운영 가능한 구조를 먼저 잡는 것이 현실적입니다.

3.2 다국어(한/영/일) — 결제까지 원하는 건지 확인 필요

RFP에 다국어(한/영/일)가 필수로 잡혀 있고, 전체 사용자 시스템에 걸쳐 있습니다. 그런데 다국어가 UI 번역인지, 결제까지 포함인지에 따라 복잡도가 완전히 달라집니다.

구분시나리오 A: 국내 외국인 UI 지원시나리오 B: 해외 거주 외국인 결제
결제 국내 PG 일반결제 + 간편결제 그대로 해외특약 별도 계약, 3DS 인증 기반 해외카드 결제
간편결제 네이버페이, 카카오페이 사용 가능 네이버페이(한국 휴대폰 필수), 카카오페이(외국인등록증+한국 휴대폰) → 사실상 불가
수수료 국내 일반 수수료 해외카드 결제 수수료 3.8~4% 수준
난이도 UI만 i18n 처리 → 낮음 PG사별 지원 범위 확인 + 해외특약 → 높음
시나리오 A는 기본 범위에 포함하고, 시나리오 B는 해외특약 계약 조건 및 PG사별 지원 범위 확인 후 추가 범위로 분리합니다.

3.3 셀러센터 다국어 — 쇼핑몰일 때 오히려 복잡도가 높다

FO(고객용) 다국어는 당연합니다. 상품명, 설명, 카테고리, 검색 등 고객 접점 전체를 i18n 처리해야 합니다.

그런데 셀러센터 다국어는 다른 문제입니다. 입점하는 셀러들이 한국 업체라면 셀러센터를 영어/일본어로 만들 이유가 없습니다. 만약 해외 셀러 입점까지 고려하는 거라면 상품 등록 시 다국어 입력 필드, 셀러-관리자 커뮤니케이션의 언어 문제, 정산서/세금계산서의 언어 등이 추가되면서 복잡도가 급격히 올라갑니다.

KT alpha 판촉몰 운영 경험에서 셀러들은 상품 등록 UI의 복잡도에 굉장히 민감합니다. 옵션 설정, 칼선 등록, 가격 매트릭스만으로도 충분히 복잡한데, 여기에 다국어 입력 필드까지 붙이면 등록 이탈률이 높아질 수 있습니다.

FO(고객용)만 다국어 처리하고, 셀러센터/BO는 한국어 단일 언어로 가는 것을 1차 범위로 잡습니다. 상품 데이터 구조는 처음부터 다국어 필드를 설계해두되, 셀러 입력 화면은 한국어로 시작하고 번역은 관리자 측에서 처리하는 방식이 현실적입니다.
