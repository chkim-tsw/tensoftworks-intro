5-1. 정산 시스템 수정 — PG 금액 불일치 대응
쿠폰/포인트 도입 시 PG 승인금액과 실제 판매금액이 달라지는 구조적 문제

현재 (AS-IS)

주문금액 = PG 승인금액
1:1 단순 매칭

판매금액 = PG결제
→

변경 후 (TO-BE)

주문금액을 3원 분리 구조로 전환
각 항목별 정산 데이터 분리

판매금액 = PG결제 + 쿠폰할인 + 포인트사용

정산 로직 변경

  • 정산 대사 로직 — PG 승인금액과 이맥스 판매금액 차이를 자동 보정
  • 정산 리포트 개편 — 쿠폰할인, 포인트사용, 실 PG결제, 순매출 컬럼 추가
  • 비용 부담 구분 — 쿠폰 할인분의 운영사/공급사 부담 비율 분리

취소/환불 시 역보정

  • PG 환불 처리 — 실 PG결제 금액만 PG사에 환불 요청
  • 쿠폰 복원 — 유효기간 내 쿠폰 자동 복원, 만료 시 소멸
  • 포인트 반환 — 사용 포인트 잔액 복원 처리
  • 정산 데이터 역산 — 취소 건에 대한 정산 역보정 자동 처리
쿠폰/포인트 시스템은 기존 이맥스 정산 구조에 직접적인 영향을 미칩니다. PG 승인금액과 판매금액의 차이가 발생하는 시점부터 정산 대사가 맞지 않으므로, 정산 시스템 수정은 쿠폰/포인트 개발과 동시에 진행되어야 합니다.

9. 핵심 요약
1

쿠폰과 포인트는 독립 시스템

정책 엔진을 통해 상호 제약 관계를 관리합니다.

2

쇼핑몰 자체 할인이 존재하므로

추가 쿠폰은 허용하되 포인트 사용은 제한하는 방향이 운영사 손해를 최소화합니다.

3

포인트는 현금성 자산

마이너스 발행 트리거와 발행 한도 등 운영사 보호 장치가 반드시 필요합니다.

4

이맥스 연동 시 핵심 기술 과제

데이터 정합성, 동시성 처리, 롤백 보장이 핵심입니다.

5

정산 시스템 수정 필수

쿠폰/포인트 도입 시 PG 승인금액 ≠ 판매금액이므로 3원 분리 구조 전환이 필수입니다.
