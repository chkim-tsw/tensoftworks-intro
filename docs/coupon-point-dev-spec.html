쿠폰 시스템 기능 정의

쿠폰 시스템은 발행 → 배포 → 사용 → 소멸의 4단계 수명 주기를 가지며, 정액/비율/무료배송/사은품 4개 유형을 지원합니다. 각 기능은 독립 모듈로 설계하여 정책 엔진과 연동됩니다.

기능 ID기능명상세 설명우선순위
쿠폰 발행 · 관리
CPN-001쿠폰 생성유형(정액/비율/무료배송/사은품), 할인값, 유효기간, 사용조건 설정필수
CPN-002발행 한도 관리총 발행 수량 상한, 1인당 최대 보유 수량 제한필수
CPN-003쿠폰 배포자동 다운로드, 코드 입력, 관리자 수동 지급 3가지 채널필수
CPN-004쿠폰 유효기간발행일 기준 / 다운로드일 기준 택1, 만료 시 자동 소멸필수
CPN-005사용 조건 설정최소 주문금액, 적용 카테고리/상품 지정, 회원등급 제한필수
쿠폰 적용 · 결제
CPN-006쿠폰 적용결제 시 사용 가능 쿠폰 자동 조회, 최적 쿠폰 추천필수
CPN-007중복 사용 정책1주문 1쿠폰 원칙, 정책 엔진 연동하여 중첩 여부 판단필수
CPN-008비용 부담 구분쿠폰별 비용 부담 주체(운영사/공급사) 필드 설계 및 정산 반영높음
취소 · 환불
CPN-009전체 취소 시 복원유효기간 내 쿠폰 자동 복원, 만료 시 소멸 처리필수
CPN-010부분 취소 처리부분 취소 시 쿠폰 미복원(기사용), 할인 혜택 유지필수
CPN-011관리자 수동 환불어드민에서 건별 쿠폰 복원 여부 선택 가능높음
어드민 관리
CPN-012쿠폰 현황 대시보드발행/사용/소멸 통계, 비용 분석, 기간별 조회높음
CPN-013쿠폰 코드 체계이맥스 연동용 고유 접두사(EMX-CPNXXXX) 코드 생성높음
설계 원칙: 쿠폰은 "1주문 1쿠폰" 원칙을 기본으로 하되, 정책 엔진 설정에 따라 쇼핑몰 자체 할인과의 중첩은 최대 할인율 제한 내에서 허용합니다.

포인트 시스템 기능 정의

포인트는 현금성 자산이므로 운영사 손해 방지를 위한 마이너스 발행(차감) 트리거와 발행 한도 설계가 핵심입니다. FIFO(선입선출) 원칙으로 먼저 적립된 포인트부터 사용·소멸됩니다.

기능 ID기능명상세 설명우선순위
포인트 적립
PNT-001구매 적립결제금액 기준 비율 적립, 구매 확정 시(취소 기간 경과 후) 지급필수
PNT-002이벤트 적립프로모션, 출석체크, 리뷰 작성 등 조건 충족 시 즉시 적립높음
PNT-003관리자 수동 지급CS 보상, 프로모션 등 관리자 승인 후 수동 지급필수
PNT-004삼성 연동 적립삼성 구독 서비스 확정 데이터 기반 후적립 (CSV/수동)중간
포인트 사용 · 결제
PNT-005포인트 결제결제 수단에 "포인트" 항목 추가, 실결제금액과 분리 표기필수
PNT-006최소 결제 비율포인트 전액 결제 방지, 최소 현금 결제 비율 강제 (예: 30%)필수
PNT-007잔액 실시간 조회마이페이지 및 어드민에서 API 기반 실시간 잔액 확인필수
차감 · 마이너스 발행
PNT-008주문 취소 차감구매 적립 포인트 자동 회수 (주문 취소 이벤트 트리거)필수
PNT-009부분 반품 차감반품 금액 비례 포인트 자동 차감필수
PNT-010오적립 보정시스템 오류 과다 적립 시 관리자 수동 차감 (사유 입력 필수)높음
PNT-011부정 사용 적발어뷰징/다중 계정 감지 시 전액 차감 + 계정 제재높음
소멸 · 관리
PNT-012자동 소멸적립일 기준 12개월(운영사 설정 가능), FIFO 순서 적용필수
PNT-013소멸 예고 알림소멸 30일 전 / 7일 전 자동 안내 (SMS/푸시)높음
PNT-014포인트 대시보드적립/사용/차감/소멸 통계, 월간 발행량 모니터링, 한도 관리높음
핵심 원칙: 포인트는 현금성 자산이므로 일일 적립 한도, 월간 발행 한도를 설정하고, 한도 초과 시 자동 중단 + 관리자 알림이 반드시 구현되어야 합니다.

정책 엔진 기능 정의

정책 엔진은 쿠폰·포인트·쇼핑몰 자체 할인 간의 중첩 규칙, 최대 할인 캡, 운영사 보호 트리거를 통합 관리합니다. 결제 시점에 6단계 우선순위로 정책이 순차 적용됩니다.

기능 ID기능명상세 설명우선순위
할인 중첩 정책
POL-001쇼핑몰 할인 + 쿠폰허용 (최대 할인율 제한 적용)필수
POL-002쇼핑몰 할인 + 포인트불가 (현재 정책, 운영사 설정 변경 가능)필수
POL-003쿠폰 + 포인트정책 설정에 따라 결정 (어드민에서 ON/OFF)필수
POL-004쿠폰 + 쿠폰불가 (1주문 1쿠폰 원칙)필수
손해 방지 트리거
POL-005최대 할인율 캡상품 원가 대비 총 할인이 설정 비율 초과 시 자동 차단필수
POL-006일일 포인트 적립 한도1인당 일일 적립 상한 도달 시 추가 적립 중단필수
POL-007월간 포인트 발행 한도전체 발행량이 월 예산 초과 시 자동 중단 + 관리자 알림필수
POL-008쿠폰 마진 체크쿠폰 적용 후 상품 마진이 기준 이하 시 적용 불가높음
결제 시 적용 순서 (6단계)
POL-009정책 우선순위 엔진① 상품 원가 확인 → ② 쇼핑몰 자체 할인 → ③ 쿠폰 적용 판단 → ④ 포인트 사용 판단 → ⑤ 총 할인율 캡 검증 → ⑥ 최종 결제금액 확정필수

EMX-003정산 반영쿠폰 할인분 정산 시 정확히 차감, 항목 분리 표기필수
EMX-004취소 웹훅이맥스 취소 API 호출 시 웹훅으로 쿠폰 상태 자동 업데이트높음
이맥스 포인트 연동
EMX-005적립 시점 동기화이맥스 주문 확정 이벤트 기준 포인트 적립필수
EMX-006포인트 결제 수단이맥스 결제 수단에 "포인트" 항목 추가, 실결제금액 분리필수
EMX-007취소 시 포인트 복원취소 유형별 포인트 복원 룰 사전 정의 및 자동 처리필수
EMX-008관리자 차감 기능이맥스 어드민에서 사유 입력 필수 + 승인 프로세스높음
공통 연동
EMX-009데이터 정합성 검증정기 교차검증 배치 (쿠폰/포인트 잔액 vs 이맥스 데이터)필수
EMX-010동시성 처리비관적 잠금 또는 큐 기반 처리로 데이터 충돌 방지필수
EMX-011트랜잭션 롤백결제/취소 과정 트랜잭션 보장, 실패 시 전체 롤백필수
EMX-012로그 보관최소 3년 보관 (전자상거래법 준수)필수

개발 로드맵

정책 엔진이 먼저 설계되어야 쿠폰/포인트 시스템의 제약조건이 확정됩니다. 정책 설계 없이 개발을 시작하면 대규모 수정이 필요할 수 있으므로, 아래 순서를 준수합니다.

순서단계주요 과업기간
1순위정책 엔진 설계할인 중첩 규칙 정의, 최대 할인 캡 설계, 운영사 보호 룰 확정, DB 스키마 설계2주
2순위쿠폰 시스템 코어발행/배포/사용/소멸 전주기 개발, 4개 쿠폰 유형, 취소/환불 로직, 어드민 관리3~4주
3순위포인트 시스템 코어적립/사용/차감/소멸 전주기, 마이너스 발행, FIFO 소멸, 잔액 조회3~4주
4순위이맥스 어드민 연동현황 대시보드, 정산 데이터 반영, 관리자 차감 기능, 웹훅 연동2~3주
5순위삼성 연동 포인트삼성 데이터 수신 프로세스, CSV 업로드, 유입 태그 자동 쿠폰 발행1~2주
총 개발 기간: 약 12~15주 (3~4개월). 1~3순위 병렬 진행 시 약 8~10주로 단축 가능. QA 및 안정화 기간 2주 별도.
